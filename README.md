# Database-First Commerce Suite

واجهة أمامية جديدة مبنية من الصفر وفق عقود البيانات المرسلة.

## المبادئ

- القراءة من الـ views الرسمية فقط.
- لا يوجد احتساب للخصومات أو التسعير في الواجهة.
- السعر النهائي يُعرض كما يُعاد من `v_runtime_products.final_price`.
- عمليات الكتابة تمر عبر routes التنفيذية المحددة في المواصفة.
- الفواتير الواتساب تُنشأ من snapshots محفوظة فقط.

## التشغيل

1. انسخ `.env.example` إلى `.env` واضبط القيم.
2. افتح الملف `index.html` عبر خادم HTTP محلي.
   - مثال: `python -m http.server 4173`
3. افتح `http://localhost:4173`

## البنية

- `src/api` عميل الاتصال وطبقة الوصول للبيانات.
- `src/screens` شاشات المتجر، الأدمن، المندوب، والعميل.
- `src/services` خدمات الجلسة والفواتير والروابط التنفيذية.
- `src/state` الحالة العامة.
- `src/utils` وظائف مساعدة عامة.

## ملاحظات تنفيذية

- في حال عدم توفر backend مطابق للعقود، ينتقل التطبيق إلى وضع demo محلي محدود للعرض فقط.
- لا يتم حساب السعر النهائي في JavaScript.
- لا يتم استنتاج الشرائح أو خصوماتها في الواجهة.
